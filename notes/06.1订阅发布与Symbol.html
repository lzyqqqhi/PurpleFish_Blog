        # 订阅发布与Symbol

        # 什么是 Symbol

        • Symbol 是 ECMAScript 6（ES6）引入的一种新的原始数据类型，表示独一无二的值，是 JavaScript 中第七种数据类型。它可以用于创建对象的唯一属性键。例如：
          const sym1 = Symbol();
const sym2 = Symbol();
console.log(sym1 === sym2); // false
          每次调用 Symbol() 函数都会创建一个唯一的值，这使得它非常适合用作对象属性的键来避免属性名冲突。

        # Symbol 的作用

        • 创建唯一属性键

          • 在对象中，如果使用普通的字符串作为属性键，可能会出现属性名冲突的情况。例如，当你创建一个对象库或者使用第三方库时，不同的库可能会使用相同的字符串作为属性键，从而导致数据被意外覆盖。而使用 Symbol 就可以有效避免这种情况。例如：
            const obj = {};
const sym3 = Symbol('name');
const sym4 = Symbol('name');
obj[sym3] = 'Alice';
obj[sym4] = 'Bob';
console.log(obj); // { [Symbol(name)]: "Bob", [Symbol(name)]: "Alice" }
            尽管 sym3 和 sym4 都使用了相同的描述字符串 'name'，但是它们是不同的键，所以对象的两个属性可以共存。

        • 定义对象的私有属性

          • 虽然 JavaScript 没有真正的私有属性，但是可以通过 Symbol 来模拟私有属性。由于 Symbol 类型的属性键不容易被发现和枚举，所以在一定程度上可以起到私有属性的作用。例如：
            const _name = Symbol('name');
class Person {
  constructor(name) {
    this[_name] = name;
  }
  getName() {
    return this[_name];
  }
}
const person = new Person('John');
console.log(person.getName()); // "John"
            这里的 _name 属性键在类的外部很难直接访问，从而模拟了私有属性的效果。

        • 定义对象的系统级内置属性

          • 例如，Symbol.iterator 可以用于定义对象的默认迭代器方法。当你使用 for...of 循环遍历一个对象时，会调用该对象的 Symbol.iterator 方法。例如：
            const myIterable = {
  [Symbol.iterator]() {
    let count = 0;
    return {
      next() {
        count++;
        if (count <= 3) {
          return { done: false, value: count };
        } else {
          return { done: true };
        }
      }
    };
  }
};
for (const num of myIterable) {
  console.log(num); // 1 2 3
}

        # Symbol 与发布订阅的联系

        • 作为事件类型标识符

          • 在发布订阅模式中，通常会使用字符串作为事件类型来标识不同的事件。但是使用字符串可能会导致事件类型冲突的问题。这时可以使用 Symbol 来定义唯一的事件类型。例如：
            const EVENT_LOGIN = Symbol('login');
const EVENT_LOGOUT = Symbol('logout');
class EventEmitter {
  constructor() {
    this.subscribers = {};
  }
  subscribe(event, callback) {
    if (!this.subscribers[event]) {
      this.subscribers[event] = [];
    }
    this.subscribers[event].push(callback);
  }
  publish(event, data) {
    if (this.subscribers[event]) {
      this.subscribers[event].forEach(callback => callback(data));
    }
  }
}
const emitter = new EventEmitter();
emitter.subscribe(EVENT_LOGIN, (user) => {
  console.log(`${user} logged in`);
});
emitter.publish(EVENT_LOGIN, 'Alice'); // 输出 "Alice logged in"
            这里使用 Symbol 来定义事件类型 EVENT_LOGIN 和 EVENT_LOGOUT，避免了使用字符串可能出现的事件类型冲突问题。

        • 隐藏发布订阅的实现细节

          • 在实现发布订阅模式的时候，可以使用 Symbol 来隐藏一些内部的属性和方法。例如，将存储订阅者的数组用 Symbol 作为键，这样外部代码就很难直接访问这个数组，从而保护了发布订阅的内部实现。例如：
            class EventEmitter {
  constructor() {
    this[Symbol.for('subscribers')] = {};
  }
  subscribe(event, callback) {
    if (!this[Symbol.for('subscribers')][event]) {
      this[Symbol.for('subscribers')][event] = [];
    }
    this[Symbol.for('subscribers')][event].push(callback);
  }
  publish(event, data) {
    if (this[Symbol.for('subscribers')][event]) {
      this[Symbol.for('subscribers')][event].forEach(callback => callback(data));
    }
  }
}
const emitter = new EventEmitter();
emitter.subscribe('login', (user) => {
  console.log(`${user} logged in`);
});
emitter.publish('login', 'Bob'); // 输出 "Bob logged in"
            这里通过 Symbol.for('subscribers') 创建了一个全局共享的 Symbol 值，用于存储订阅者信息，外部很难直接通过常规方式访问这个存储结构，从而隐藏了发布订阅的实现细节。
