# DOM 树、CSSOM 树加载与渲染原理，重排/重绘概念及优化方案

# 一、DOM 树与 CSSOM 树加载和渲染原理

# 1. 什么是 DOM 树和 CSSOM 树？

  • DOM（Document Object Model）树

    • 浏览器解析 HTML，按标签层级结构生成的一棵“文档对象树”。
    • 反映页面的结构和内容。

  • CSSOM（CSS Object Model）树

    • 浏览器解析 CSS、内联样式、style 标签及外链样式，构建出的“样式对象树”。
    • 反映页面各元素的样式层级和计算规则。

# 2. 加载与渲染流程（核心步骤）

  1. 浏览器自上而下解析 HTML，构建 DOM 树。
  2. 遇到 <link rel="stylesheet"> 或 <style> 时，阻塞渲染，开始加载/解析 CSS，构建 CSSOM 树。
  3. 当 DOM 树和 CSSOM 树都构建完成后，合并为 Render Tree（渲染树）。
    1. 渲染树只包含可见节点，且每个节点携带最终计算样式。
  4. 浏览器根据渲染树，计算元素几何信息（布局），并进行后续绘制。

# 3. 渲染示意图

HTML（DOM树） + CSS（CSSOM树）
          ↓
     渲染树(Render Tree)
          ↓
        布局（Layout）
          ↓
        绘制（Paint）

# 二、重排（Reflow）与重绘（Repaint）概念

# 1. 重排（Reflow）

  • 定义：当页面元素的结构或几何信息发生变化（如：尺寸、位置、内容等），浏览器会重新计算内容及其子树的布局。
  • 触发时机：添加/删除节点、改变元素大小/位置、修改 display/width/height/margin/padding/边框等。
  • 特点：开销大、影响范围可能很广，可能导致整棵渲染树重新计算。

# 2. 重绘（Repaint）

  • 定义：当元素某些外观样式变化时（如颜色、背景、可见性等），只需要重新绘制外观，不影响布局。
  • 触发时机：更改 background、color、visibility、box-shadow 等样式。
  • 特点：只影响外观，开销小于重排。

# 三、常见优化方案

优化目标 具体策略
减少重排 - 批量操作 DOM（离线节点、Fragment）
- 避免频繁逐一修改样式
- 使用 class 控制批量样式切换
- position 定位脱离文档流的元素
避免频繁 DOM 查询和布局 - 缓存查询结果
- 避免在循环/动画中频繁读写 DOM 属性
- 用 requestAnimationFrame 合理批量处理动画
合理触发 GPU 合成 - 动画使用 transform、opacity 而不是 top/left/width/height 等属性
降低重绘范围 - 分层绘制（如主内容和动画层分离）
- 尽量只重绘必要区域

# 四、总结口诀&记忆

DOM 树结构，CSSOM 管样式，渲染树合体再绘制。

重排影响大，重绘代价小，优化策略批量做，动画优用 transform！

# 五、课件总结金句
