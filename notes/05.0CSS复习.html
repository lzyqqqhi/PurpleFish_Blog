                            # CSS复习

                            # 一、CSS 的语法结构

                            CSS 的顶层样式表主要由两种规则构成：

                            1. At - rule（@规则） ：以 @ 开头的特殊规则，主要用于实现高级特性。
                            2. Qualified rule（普通规则） ：包含选择器、属性和声明区块，是在开发过程中最常使用的规则，用于日常的样式定义和应用。

                            # 二、CSS 的 At - rule（@规则）

                            以下是常见的 @规则及其用途：

                            1. @charset ：用于指定 CSS 文件的字符编码，例如 @charset "UTF - 8"; ，一般位于 CSS 文件开头，确保浏览器正确解析文件中的字符，避免因编码问题导致样式异常。
                            2. @import ：用于引入外部 CSS 文件，可结合 supports 和 media query 等条件引入不同的样式文件。如 @import "style.css"; 直接引入整个文件，@import url("theme.css") screen and (min - width: 1024px); 则在屏幕宽度大于等于 1024px 的情况下才引入指定的 CSS 文件，实现不同设备或屏幕尺寸下的样式适配。
                            3. @media ：媒体查询规则，依据设备类型（如屏幕、打印）或特性（如屏幕宽度）应用特定样式。例如 @media screen and (max - width: 600px) {body {background - color: lightblue;}} ，当屏幕宽度不超过 600px 时，body 元素的背景色变为浅蓝色，用于优化移动设备等小屏幕显示环境下的页面样式。
                            4. @page ：针对分页媒体（如打印文档）设置页面样式，如 @page {margin: 1in;} 可设置打印页面的页边距，提高打印输出效果，使文档在打印时更符合预期的版式要求。
                            5. @counter - style ：用于定义自定义列表项的样式，可定义计数系统的外观和行为。比如 @counter - style customCounter {system: numeric; symbols: "🟠" "🟡" "🟢";} 创建了一个使用特定符号表示数字的计数器样式，可用于 ordered - list 等元素的列表项标记自定义，实现独特的列表编号效果。
                            6. @keyframes ：定义动画的关键帧，用于指定动画在不同时间点的状态，从而实现各种动画效果。例如 @keyframes fadeIn {from {opacity: 0;} to {opacity: 1;}} 定义了一个从完全透明到完全不透明的渐变动画，可配合动画相关的 CSS 属性（如 animation - name、animation - duration 等）应用于元素，使元素在页面中具备动态表现力。
                            7. @font - face ：加载自定义字体，使得网页能够使用特定的字体展示文本内容。如 @font - face {font - family: "CustomFont"; src: url("customfont.woff") format("woff"); font - weight: normal;} 引入外部字体文件，并为其指定字体族名称、来源、字重等信息，之后可通过 font - family 属性在样式中引用该字体，丰富页面的字体样式选择，提升文本的视觉表现。
                            8. @supports ：用于检测浏览器是否支持某些 CSS 特性，从而实现样式的有条件应用，避免在不支持的浏览器中出现样式错误或布局问题。例如 @supports (display: grid) {div {display: grid;}} ，当浏览器支持 grid 布局时，div 元素采用 grid 布局，否则可设置其他布局方式作为替代，确保页面在不同浏览器下的兼容性和可用性。
                            9. @namespace ：与 XML 命名空间配合使用，为选择器添加特定的命名空间限定，提高样式的针对性和准确性。例如 @namespace svg url([http://www.w3.org/2000/svg](http://www.w3.org/2000/svg)); svg|circle {fill: red;} 用于处理 XML 文档中的元素，通过命名空间限定选择器范围，尤其在处理复合文档或包含多种 XML 命名空间的场景下非常有用，可有效避免样式冲突和选择器作用范围过广的问题。
                            10. @viewport ：用于定义视口配置，不过通常由 HTML 的 <meta> 标签替代。例如 @viewport {width: device - width;} 可设置视口宽度为设备宽度，对页面在移动设备上的显示布局有一定影响，使页面能够更好地适应不同设备的屏幕尺寸。
                            11. @color - profile ：用于定义颜色特性，但一般不推荐使用，在一些特定的专业颜色管理场景中可能会有一定应用。

                            # 三、CSS 的 Qualified rule（普通规则）

                            普通规则是 CSS 开发的核心部分，具体组成如下：

                            1. 选择器 ：用于定位页面中的元素，以便为其应用样式。例如 h1 {color: blue;} 中的 h1 就是一个选择器，它选中页面中所有的 h1 元素，并将其颜色设置为蓝色。通过不同的选择器类型（如元素选择器、类选择器、ID 选择器、属性选择器等），可以精准地控制样式的作用范围，实现对页面中特定元素或一组元素的样式定义。
                            2. 声明区块 ：用花括号 {} 包裹，包含具体的样式声明。如 body {font - family: Arial, sans - serif; margin: 0;} 中的花括号部分就是一个声明区块，里面存放多个属性 - 值对。声明区块的作用是为选中的元素定义一系列的样式属性和对应的值，从而综合控制元素的外观和布局，例如字体、颜色、尺寸、边距、对齐方式等方面。
                            3. 属性值 ：样式的声明由属性和对应的值组成。例如 div {width: 100px; height: 100px; background - color: red;} 中，width、height 和 background - color 是属性，100px、100px 和 red 是相应属性的值，分别设置了 div 元素的宽度、高度和背景颜色。属性用于定义元素的某个具体样式方面，而值则确定了该属性的具体表现形式，通过属性和值的组合，可以精确地指定元素的样式细节，使其在页面中呈现出预期的视觉效果和布局形态。
                            4. 值的类型和函数 ：值的类型丰富多样，除了常见的颜色（如 red、blue、#ff0000、rgb(255, 0, 0) 等）、长度（如 px、em、rem、vw、vh 等）、百分比（如 50%、100%）等基本数据类型，还可以包含函数。例如 p {font - size: calc(1em + 2px); background - image: url("background.jpg");} 中，calc() 函数用于计算字体大小，能够基于相对单位和绝对单位进行动态计算，实现更灵活的样式定义；url() 函数用于指定背景图片的路径，引入外部图像资源作为元素背景，为样式提供了更多强大且灵活的功能，使得样式可以适应不同的设计需求和布局场景。

                            # 四、CSS选择器

                            # 1. 分隔符符号

                            选择器之间的关系通过以下符号连接：

                            • 空格：表示后代选择器（优先级低）。
                            • >：表示子代选择器。
                            • +：表示相邻兄弟选择器。
                            • ~：表示一般兄弟选择器。
                            • ||：表示表单或表格内的列选择器。

                            代码示例：

                            /* 后代选择器 */
div p {
  color: blue;
}

/* 子代选择器 */
div > p {
  color: red;
}

/* 相邻兄弟选择器 */
h1 + p {
  font-size: 14px;
}

/* 一般兄弟选择器 */
h1 ~ p {
  margin-top: 10px;
}

/* 列选择器 */
|| td {
  background: lightgray;
}

                            # 选择器类型

                            选择器的基本结构包含标签类型、id、class、属性选择器和伪类。伪类可以之后追加伪元素。

                            # 1. 标签类型选择器

                            直接指定 HTML 元素名称选择对应元素。

                            代码示例：

                            /* 选择所有段落 */
p {
  line-height: 1.5;
}

                            # 2. ID 选择器

                            通过元素的唯一标识 id 进行选择（#符号）。

                            代码示例：

                            /* 选择具有 id 为header的元素 */
#header {
  background-color: lightblue;
}

                            # 3. Class 选择器

                            通过元素的类名 class 进行选择（.符号）。

                            代码示例：

                            /* 选择具有类名为highlight的元素 */
.highlight {
  color: yellow;
}

                            # 4. 属性选择器

                            通过 HTML 元素的属性值进行精准选择。

                            符号 含义 代码示例
                            [attr] 选择有指定属性的元素 input[data-required] { border: 1px solid red; }
                            [attr=value] 选择属性值匹配的元素 input[type="text"] { padding: 5px; }
                            [attr*=value] 包含指定值的属性 a[href*="https"] { color: green; }
                            [attr^=value] 属性值以指定字符开头 input[name^="user"] { background: lightgray; }
                            [attr$=value] 属性值以指定字符结尾 a[href$=".pdf"] { text-decoration: underline; }

                            # 5. 伪类选择器

                            用于选择特定状态的元素（以 : 开头）。

                            伪类 作用 代码示例
                            :hover 选中鼠标悬停状态的元素 button:hover { background: yellow; }
                            :nth-child(n) 选中其父元素的第 n 个子元素 li:nth-child(2) { font-weight: bold; }
                            :focus 选中获得焦点的元素 input:focus { border-color: blue; }
                            :not(selector) 选取非某选择器的元素 div:not(.excluded) { color: black; }

                            # 6. 伪元素选择器

                            用于选择元素的特定部分（以 :: 开头）。

                            结合清除浮动

                            伪元素 作用 代码示例
                            ::before 在元素开头生成内容 h1::before { content: "> "; }
                            ::after 在元素结尾生成内容 p::after { content: "Read more."; }
                            ::placeholder 针对表单元素占位符 input::placeholder { color: gray; }

                            # 选择器的叠加规则

                            选择器可以通过组合实现复杂的筛选功能：

                            # 1. 组合多种选择器

                            可以叠加标签类型、class、id、属性和伪类选择器。

                            代码示例：

                            /* 选择所有 class 为 special 的段落 */
p.special {
  background-color: lightyellow;
}

/* 选择具有 id 为 main 且 title 属性存在的元素 */
#main[title] {
  padding: 10px;
}

/* 同时选择 h1 和 h2 元素 */
h1, h2 {
  font-size: 20px;
}

                            # 2. 使用父子关系符号

                            通过 > 等表示父子关系或兄弟关系。

                            代码示例：

                            /* 特定父子关系 */
div > p {
  color: blue;
}

/* 选择兄弟关系的 p */
h1 + p {
  color: darkgray;
}

                            # 选择器语法总结

                            选择器主要通过 结构符号（空格、>、+、~）、基本类型（标签、class、id）、伪类 和 伪元素 实现对页面元素的精准选择。以下是重点概括：

                            • 基本符号：空格表示后代选择器，> 表示子代选择器，+ 表示相邻兄弟选择器。
                            • 选择器类型：标签类型选择器、class 选择器、ID 选择器、属性选择器、伪类选择器、伪元素选择器。
                            • 叠加与组合：选择器可以组合使用以实现复杂筛选。

                            # 五、em ‧ rem ‧ vw 的原理与实践

                            # 1. em ‧ rem ‧ vw 背后的计算规则

                            单位 参照物 优/缺点 常见场景
                            em 当前元素的 font-size 级联导致值易被放大 / 缩小 图标、按钮内边距与文字联动
                            rem 根元素 (**<html>**) 的 font-size 全局可控、避免层层叠加 响应式版面、排版网格
                            vw 视口宽度 （viewport width） 与屏幕宽度同步，无须媒体查询；但高度不参与计算 全屏横幅、流式排版、字体随屏放大

                            # 1.1 em / rem 的像素换算

                            若浏览器根字体大小为 $16\text{px}$，则

                            $$\text{rem值}=\frac{\text{设计稿 px}}{16}$$

                            示例：24 px → $24 ÷ 16 = 1.5\text{rem}$。

                            而 em 则取 父级 字体大小：

                            $$\text{em值}=\frac{\text{设计稿 px}}{\text{父元素 font-size(px)}}$$

                            /* rem-flexible.js
 * 以设计稿宽度 designWidth 为基准，把 1rem 等同于 designWidth/10 px
 * 典型做法：设计稿 375px → 1rem = 37.5px
 */
(function (doc, win) {
  const docEl = doc.documentElement;
  const designWidth = 375;          // ↰ 按需改成 750、414…
  const remBase    = 10;            // 设计稿被均分成 10 份，1rem = (vw/设计宽)×designWidth/remBase

  function setRemUnit() {
    const clientW = docEl.clientWidth || designWidth; // Fallback
    const rem = (clientW / designWidth) * (designWidth / remBase);
    docEl.style.fontSize = rem + 'px';
  }

  setRemUnit();
  win.addEventListener('resize', setRemUnit);
  win.addEventListener('pageshow', e => e.persisted && setRemUnit()); // iOS bfcache
})(document, window);

                            # 1.2 vw 的像素换算

                            公式（以 750 px 设计稿为例）：

                            $$\text{vw值}=\frac{\text{设计稿 px}}{750} \times 100\text{vw}$$

                            若设计稿宽 750 px 中有 150 px 区块：

                            $$\frac{150}{750}\times100 = 20\text{vw}$$

                            /**
 * px → vw
 * @param  {number} px           设计稿中的像素值
 * @param  {number} designWidth  设计稿宽度，默认 375
 * @param  {number} fixed        小数位，默认 4
 * @returns {string}             形如 '12.5333vw'
 */
export function px2vw(px, designWidth = 375, fixed = 4) {
  const vw = (px / designWidth) * 100;
  return vw.toFixed(fixed) + 'vw';
}

/**
 * vw → px（常用于运行时取真实像素）
 * @param {number|string} vwStr  形如 20 或 '20vw'
 * @returns {number}             真实像素
 */
export function vw2px(vwStr) {
  const num = parseFloat(vwStr);
  const viewportWidth = window.innerWidth || document.documentElement.clientWidth;
  return (num / 100) * viewportWidth;
}
// postcss.config.cjs
module.exports = {
  plugins:{
    'postcss-px-to-viewport':{
      viewportWidth:750,          // 设计稿宽
      propList:['*'],
      unitPrecision:5,
      exclude:[/node_modules/]
    }
  }
};

                            简答

                            em 的基准就是“当前层级的字体大小 (font-size)”。

                            • 当 em 用在 font-size 本身 时：

                            1 em 取 父元素 的 font-size。

                            • 当 em 用在 其他任何属性（如 padding、margin、width、height、box-shadow…）时：

                            1 em 取 当前元素 经过计算后的 font-size。

                            换句话说：

                            1. 先根据父元素字体大小求出本元素的字体大小；
                            2. 然后所有非 font-size 的 em，都拿这一步得到的“本元素字体大小”再去换算。

                            # 例子：逐级递归的计算过程

                            <style>
html  { font-size:16px; }          /* 浏览器默认 16px */
body  { font-size:1em;  }          /* 16px (取 html)   */
.box  { font-size:0.75em;          /* 12px (取 body)   */
        padding:2em; }             /* 24px (取 12px×2) */
</style>

<div class="box">内容</div>
                            1. <body> 的 font-size:1em ⇒ 1 em = 16 px ⇒ body 字号 16 px。
                            2. .box 的 font-size:0.75em ⇒ 0.75 × 16 px = 12 px。
                            3. .box 的 padding:2em ⇒ 2 × 12 px = 24 px。

                            如果 .box 里还有 .child{font-size:2em;}，则 2 em 会再乘 .box 当前的 12 px，得到 24 px，如此层层递进。

                            # 记忆口诀

                            • font-size 用 em → “向上看”（取父级）。

                            • 其他属性用 em → “看自己”（取本级）。

                            这样既能让文本大小随层级继承，又能让与文字相关的间距、圆角、图标等保持同比例缩放。

                            # 2. 代码示例：快速对比

                            <!DOCTYPE html>
<html lang="zh-CN">
<head>
<meta charset="UTF-8" />
<meta name="viewport" content="width=device-width, initial-scale=1.0" />
<title>em / rem / vw Demo</title>
<style>
/* 1. 统一根字号：10px（16×62.5%≈10） */
html { font-size: 62.5%; }

/* 2. em：相对父元素文字大小 */
.card {
  font-size: 1.6rem;   /* card 文字：16px */
  padding: 2em;        /* 随文字放大缩小 */
  background: #f3f3f3;
}

/* 3. rem：全局比例，不受父级影响 */
.title {
  font-size: 2.4rem;   /* 24px = 2.4rem */
  margin-bottom: 1rem; /* 10px */
}

/* 4. vw：视口等比缩放 */
.banner {
  width: 100vw;
  height: 25vw;        /* 高度 = 25% 屏宽 */
  background: linear-gradient(135deg,#3a7bfd,#00d2ff);
  color: #fff;
  display: flex;
  align-items: center;
  justify-content: center;
  font-size: 5vw;      /* 字体=5%屏宽 */
}
</style>
</head>
<body>
<section class="banner">VW Banner</section>

<div class="card">
  <h2 class="title">em / rem 对比</h2>
  <p>本段采用 em 内边距，随字体大小级联放大。</p>
</div>
</body>
</html>

                            复制以上文件、调整浏览器窗口宽度，即可观察：

                            1. .banner 高度与文字均随 视口宽度 实时变化；
                            2. .card 内边距随 .card 字号（em）变化；
                            3. .title 字号固定按根字号（rem）计算，无层级放大副作用。

                            # 3. 在真实项目中的用法策略

                            1. 设定基线在 <html> 设定 62.5% 或媒体查询动态 font-size，便于 rem 换算（1 rem=10 px）。
                            2. 布局用 rem，组件内微调用 em
                              1. 布局网格、间距：rem → 保持全局一致。
                              2. 组件内缘、图标与文字：em → 随文字放大保持比例。
                            3. 无需断点时用 vw / vminLanding Page、全屏幻灯片、海报式排版可直接用 vw/vh，少写媒体查询。
                            4. 兼顾可访问性用户浏览器字体放大时，rem / em 能自然放大，vw 不会；若页面核心信息用 vw，请配合 @media (prefers-reduced-motion) 或额外字号上限，确保阅读体验。

                            # 4. 一张速查“换算卡片”

                            设计稿 → 实际 CSS
----------------
PX → REM:   px ÷ 根字体(px)
PX → EM :   px ÷ 父字体(px)
PX → VW :   (px ÷ 设计稿宽) × 100vw

                            # 六、Flex布局

                            # Flex 的工作核心
                            • Flex 容器：通过设置 display: flex，元素成为 Flex 容器。
                            • Flex 项：Flex 容器中的直接子元素被称为 Flex 项。
                            • Flex 容器按照一个主轴方向排列 Flex 项。
                            • Flex 项可以利用 flex 属性动态调整自身尺寸，从而“填补剩余空间”。
                            # Flex 布局的设计原理

                            Flex 布局的设计哲学基于以下几个关键角色和属性：

                            1. Flex 容器（Flex Container）

                              1. 定义方式：设置容器 display: flex 或 display: inline-flex。
                              2. 核心功能：
                                • 将子元素视为 Flex 项，按规则排列。
                                • 提供控制轴方向、对齐方式等功能。

                            2. 轴的概念Flex 容器中有两条概念轴：

                              1. 主轴（Main Axis）：决定 Flex 项的排列方向。
                                • 默认主轴为水平方向。
                              2. 交叉轴（Cross Axis）：与主轴垂直的方向。
                                • Flex 布局允许子项在交叉轴自由排列。
                              3. 主轴方向可通过 flex-direction 调整。

                              示意图：

                            主轴方向（flex-direction 默认是 row）
------------------------------------>
| [子项1]  [子项2]  [子项3]
|
↓ (交叉轴方向)

                            1. Flex 属性每个 Flex 项通过 flex 属性控制其尺寸伸缩比例：

                              1. flex: [flex-grow] [flex-shrink] [flex-basis];
                                • flex-grow：定义剩余空间分配比例。
                                • flex-shrink：定义 Flex 项是否可以收缩，以及收缩比例。
                                • flex-basis：定义分配大小的初值。

                            2. 对齐与分布Flex 提供强大的对齐和分布控制属性：

                              1. 主轴对齐（**justify-content**）：

                              2. 控制 Flex 项在主轴的排列方式。

                                • 常用值：
                                  • flex-start：从起点开始排列（默认）。
                                  • center：主轴居中。
                                  • space-between：两端对齐，项目之间平均分配空间。
                                  • space-around：每个项目两侧留空间，空间均等。
                            • 交叉轴对齐（**align-items**** 和 **align-self**）**：

                            控制 Flex 项在交叉轴上的位置。

                            • 常用值：

                              • stretch：拉伸到填满整个容器高度（默认）。
                              • center：在交叉轴居中。

                            • 多行对齐（**align-content**）：

                            如果 Flex 项存在多行，控制各行之间的间距。

                            # Flex 布局的应用场景

                            Flex 布局在现代应用中非常普遍，尤其适用于动态和响应式布局。

                            1. 垂直居中

                            通过 Flex 布局，可以轻松实现水平和垂直居中。

                            代码示例：

                            .container {
  display: flex;
  justify-content: center; /* 水平居中 */
  align-items: center;    /* 垂直居中 */
  height: 100vh;          /* 满屏高度 */
  background: lightgrey;
}
.item {
  width: 100px;
  height: 100px;
  background: lightblue;
}

                            效果示意：

                            • 一个子项在屏幕中央完美居中。

                            2. 两列等高布局

                            Flex 轻松解决“列内容高度动态变化时保持等高”的问题。

                            代码示例：

                            .container {
  display: flex;
}
.column {
  flex: 1; /* 每列平分宽度 */
  background: lightblue;
}

                            效果示意：

                            • 两列布局，即使其中一个列内容较多，任然保持等高。

                            3. 自适应宽度布局

                            使用 flex-grow 实现剩余空间的动态分配。

                            代码示例：

                            .container {
  display: flex;
}
.item {
  flex: 1; /* 均分所有剩余空间 */
  background: lightgreen;
}
.item:nth-child(2) {
  flex: 2; /* 第二项占两倍宽度 */
}

                            效果示意：

                            • 子项根据 flex 属性动态调整宽度。

                            # 九、BCF

                            # 一、什么是 BFC？

                            • BFC：Block Formatting Context，块级格式化上下文。
                            • 概念：BFC 是页面中的一个独立渲染区域，区域内的盒子布局不会影响到区域外的元素，同时它自身也不会被外部的布局影响。
                            • 作用：用于解决常见布局问题，如清除浮动、阻止外边距重叠、实现两栏自适应布局等。

                            # 二、如何触发（创建）BFC？

                            常见的触发 BFC 的方式有：

                            方式 示例代码
                            根元素 <html> 页面默认
                            浮动元素 float 不为 none float: left/right;
                            绝对定位 position 为 absolute/fixed position: absolute;
                            display 为 inline-block、table-cell、flex、grid display: inline-block;
                            overflow 为非 visible overflow: hidden;
                            contain: layout; contain: layout;

                            常用触发方法代码示例：

                            .bfc {
  overflow: hidden;      /* 常用 */
  /* 或 */
  float: left;
  /* 或 */
  display: inline-block;
  /* 或 */
  position: absolute;
}

                            # 三、BFC 的作用和应用场景

                            1. 清除浮动容器设置 BFC，可以自动包裹内部浮动元素，解决高度塌陷问题。

                            2. **阻止外边距重叠（合并）**相邻块级元素的垂直外边距会重叠。通过 BFC 可以阻止这种重叠。

                            3. 自适应两栏布局利用 BFC 实现两栏自适应，边栏 float 主区触发 BFC 不受影响。

                            # 四、经典例题

                            • 为什么有时给元素加 overflow: hidden *可以解决布局异常？*因为这触发了 BFC，使元素成为独立的布局区域，防止外部影响。

                            # 五、总结记忆口诀

                            • 触发 BFC 方法要记牢：
                              • 设置 overflow、float、position、display 即可。
                            • 主要作用请牢记：
                              • 清除浮动
                              • 阻止外边距合并
                              • 复杂自适应布局

                            一句话概括：

                            BFC 是 CSS 布局中的“隔离墙”，巧用它能彻底解决常见的浮动、外边距重叠等难题，是前端布局的重要基础知识。

                            # 盒模型

                            • 盒模型是 CSS 布局的基础，每一个元素在页面上都被当作一个矩形的“盒子”。
                            • 盒子的组成：
                              • 内容区（content）：实际内容
                              • 内边距（padding）：内容和边框之间的区域
                              • 边框（border）：包裹内容和内边距的线
                              • 外边距（margin）：盒子与其他元素之间的间距

                            # 可视化结构

                            +-------------------------------+
|         margin                |
|  +------------------------+   |
|  |      border            |   |
|  |  +-----------------+   |   |
|  |  |   padding       |   |   |
|  |  |  +----------+   |   |   |
|  |  |  | content  |   |   |   |
|  |  |  +----------+   |   |   |
|  |  +-----------------+   |   |
|  +------------------------+   |
+-------------------------------+

                            # 二、盒模型的分类

                            # 1. 标准盒模型（content-box）

                            • width 和 height 代表内容区大小。
                            • 实际盒子总宽度 = $$content + padding + borde$$
                            • 代码设置： 
                              • box-sizing: content-box; /* 默认值 */

                            # 2. IE盒模型（border-box）

                            • width 和 height 包含内容区、内边距、边框的总宽度和总高度。
                            • 代码设置： 
                              • box-sizing: border-box;

                            # 盒模型宽度公式对比

                            • 标准盒模型（content-box）：
                              • $$\text{width (实际总宽)} = \text{content} + \text{padding} + \text{border}$$
                            • IE盒模型（border-box）：
                              • $$\text{width (实际总宽)} = \text{width属性设定值}$$

                            # 三、实际开发中的应用

                            • 推荐统一使用 border-box，防止盒子尺寸超出父级，方便布局和维护。
                            • 全局设置方法： 
                              • * {
  box-sizing: border-box;
}

                            # 四、拓展：特殊影响

                            • 盒模型影响width/height实际显示效果。
                            • 默认所有元素是 content-box。
                            • 盒模型影响 margin 重叠、嵌套、浮动等布局相关问题。

                            # 五、总结口诀

                            • 内容、内边距、边框、外边距，盒模型四层你要清；
                            • content-box 看内容，border-box 箱全齐！

                            一句话概括：

                            盒模型是 CSS 布局的核心，每个页面元素都由内容、内边距、边框和外边距组成，理解并合理设置盒模型，是实现精准布局的基础。
