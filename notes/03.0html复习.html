                            # HTML复习

                            # DOM

                            1. 网页的基本结构

                            • 声明文档类型（<!DOCTYPE html>），这是告诉浏览器该文档是 HTML5 文档。

                            • <html> 元素是所有其他 HTML 元素的根元素，它有 lang 属性，用于指定页面的主体语言，例如 <html lang = "zh - CN"> 表示页面的主要语言是简体中文。

                            • <head> 元素包含文档的元（meta）信息，这些信息不会显示在页面上，但对于页面的正确显示和搜索引擎优化等非常重要。

                              • <meta>标签用于定义网页的元数据，包括字符编码（<meta charset = "UTF - 8">）、视口设置（<meta name = "viewport" content = "width = device - width, initial - scale = 1.0">）、网页描述（<meta name = "description" content = "网页的简要描述">）等。

                              • <title> 标签定义文档的标题，它会显示在浏览器的标签页上。

                            • <body> 元素包含用户可以看到的所有内容，如文本、图像、链接、列表、表格等。

                            1. DOM 树的属性
                            • DOM（文档对象模型）树是一个以树形结构表示的网页内容模型，它将网页的各个元素表示为对象。
                            • 每个节点都可以看作是一个对象，节点对象有多种属性：
                              • 节点类型相关属性
                                • nodeType：返回节点的类型，不同类型的节点返回不同的值，例如元素节点返回 1，属性节点返回 2，文本节点返回 3 等。
                                • nodeName：返回节点的名称，对于元素节点来说，是元素的标签名；对于属性节点来说，是属性的名称。
                              • 节点内容相关属性
                                • nodeValue：对于某些节点类型（如文本节点、属性节点），它表示节点的值，例如文本节点的值就是其包含的文本内容，属性节点的值就是该属性的值，但对于元素节点来说，该属性通常为null。
                                • textContent：获取或设置节点及其子节点的文本内容，包括元素节点的所有文本内容（会忽略标签），这与innerText类似，但在某些细节上有区别，比如textContent会包含隐藏元素的文本内容。
                              • 节点关系相关属性
                                • parentNode：获取当前节点的父节点，用于向上遍历文档树。
                                • childNodes：以一个类似数组的对象（NodeList）返回当前节点的所有子节点，这些子节点包括元素节点、文本节点等。
                                • firstChild：获取当前节点的第一个子节点。
                                • lastChild：获取当前节点的最后一个子节点。
                                • previousSibling：获取当前节点的上一个兄弟节点，兄弟节点是指有相同父节点的节点。
                                • nextSibling：获取当前节点的下一个兄弟节点。
                              • 元素节点特定属性
                                • tagName：对于元素节点，返回该元素的标签名。
                                • attributes：返回一个包含该元素所有属性的集合（NamedNodeMap），每个属性都是一个对象，可以通过属性名来访问。例如，element.attributes.getNamedItem("id").value可以获取元素的 id 属性值。

                            # HTML5与语义化

                            1. 标题元素
                            • <h1> - <h6>：用于定义标题，<h1> 是最高级标题，<h6> 是最低级标题，文本大小依次减小，通常用于网页中大大小小的标题部分，帮助组织内容结构，让用户和搜索引擎都能快速了解页面内容的层次关系。
                            1. 段落元素
                            • <p>：用于定义段落，把内容组织成一段一段的文本，使页面上的文字内容更加清晰可读，是网页文本内容最基本的语义化容器之一。
                            1. 列表元素
                            • 无序列表<ul>和列表项<li>：<ul> 用于创建无序列表，<li> 定义列表项，适用于列举没有先后顺序或重要程度之分的项目，比如一个产品特点列表等。
                            • 有序列表<ol>和列表项<li>：<ol> 创建有序列表，列表项会自动编号，常用于步骤说明、任务列表等有顺序要求的内容。
                            • 定义列表<dl>、定义项目<dt>和定义描述<dd>：<dl> 创建定义列表，<dt> 指明被定义的项目或术语，<dd> 提供对应的解释或描述，常用于术语表、属性值说明等场景，例如对一些专业术语的解释。

                            1. 语义化块级元素

                              html5

                            • <header>：定义文档或节的页眉，通常包含标题、导航链接等，是页面顶部区域或文章开头部分的标志性元素，提供页面或内容区块的入口信息。

                            • <footer>：定义文档或节的页脚，一般位于页面底部或文章末尾，包含版权信息、链接、作者信息等，用于补充页面或内容的附加信息和底部导航。

                            • <nav>：定义页面的导航链接部分，里面通常包含一系列的链接，用于帮助用户在网站的不同页面或部分之间进行导航，如主导航栏、侧边栏导航等。

                            • <article>：独立的内容块，如文章、评论、帖子等，可以独立于文档其他部分进行分发或重用，强调了内容的完整性与独立性，让开发者能够清晰地定义主要的、核心的业务内容。

                            • <section>：文档或应用中的一个区域，用于在文档中划分不同部分或主题，一般包含标题和相关的内容，比<div>更具语义化，有助于组织结构化的内容，例如一个章节或多个功能模块的分组。

                            • <aside>：侧边栏内容，包含与页面内容相关的附加信息，如广告、侧边栏导航、推荐内容等，这些内容对理解主要内容不是必需的，但提供了额外的参考或拓展信息。

                              html5

                            • <blockquote> ：标签用于定义长的引用内容，通常是较长的文章片段或段落引用，表明被引用的内容与当前内容有某种引用关系，比如引用他人的观点、文章片段等。

                            • <pre>：标签用于定义预格式化文本，表示文本的格式（包括空格、换行等）需要被保留，通常用于展示代码、诗歌、地址等对格式要求严格的文本内容。

                            1. 语义化行内元素
                            • <strong>：表示文本的重要性，强调内容具有重要的意义，通常显示为粗体，提醒用户重点关注这部分内容，在语义上比单纯的粗体标签更明确。
                            • <em>：表示强调的文本，在语义上强调文本情感或语气，通常会以斜体显示，用于引起读者对某个词或短语的注意，比如在句子中突出某个关键词。
                            • <mark>：用于高亮显示文本，通常在视觉上会以黄色背景等醒目方式突出显示，表示文本内容已经添加了高亮标记，用于标记需要特别注意的部分，如文档中的高亮段落或关键词。
                            • <code>：定义计算机代码，用于展示代码片段或编程相关的文本，如 HTML、CSS、JavaScript 等代码，浏览器会以等宽字体显示，使代码格式更清晰，便于阅读和区分。
                            • <cite>：表示引用的作品标题，如书籍、文章、电影等名称，通常以斜体显示，用于标明引用来源或提到某部作品时的格式化元素。
                            • <q>：用于较短的引用，其内容会自动添加引号，语义上明确表示引用了他人的话语，有助于文本的准确性和规范性，浏览器会根据语言环境自动添加合适的引号样式。
                            1. 多媒体元素
                            • <audio>：用于嵌入音频文件，如音乐或其他音频流，提供了播放、暂停、音量控制等基本功能。
                            • <video>：用于嵌入视频文件，如电影片段或其他视频流，支持播放控制和全屏等功能。
                            • <source>：用于为 <picture>、<audio> 和 <video> 元素指定媒体资源，可以指定多个媒体资源，由浏览器选择最合适的资源进行加载。
                            • <track>：为 <audio> 和 <video> 元素定义外部文本轨道，如字幕、章节、描述等。
                            1. 图像相关元素
                            • <canvas>：用于绘制图形，通过脚本（通常是 JavaScript）来绘制图像、图表、动画等，提供了像素级的控制。
                            • <svg>：用于定义矢量图形，SVG 图像在放大时不会失去清晰度，适用于图标、图表等需要高质量图形的场景。
                            • <picture>：用于包含多个 <source> 元素和一个 <img> 元素，根据不同的条件（如屏幕尺寸、设备像素比等）选择最合适的图像资源进行加载。
                            1. 表单相关元素
                            • <datalist>：为 <input> 元素提供预定义的选项列表，用户可以在输入时从下拉列表中选择选项，提高了输入效率。
                            • <detail>：创建用户可以查看或隐藏的额外信息，通常用于显示不重要的或次要内容，如帮助信息、额外说明等。
                            • <dialog>：定义对话框或其他弹出对话框，如提示框、警告框等，用于实现用户与页面之间的交互。
                            • <output>：表示不同类型的输出，如脚本的输出、计算结果等，通常用于显示动态计算或操作的结果。
                            • <progress>：表示任务的进度条，用于显示某个过程的完成情况，如文件上传进度、任务执行进度等。
                            • <meter>：表示某个范围内的数值，如磁盘使用情况、投票结果等，用于显示一个定量的度量值。
                            1. 其他语义化元素
                            • <command>：定义命令按钮，如单选按钮、复选框、按钮等，用于执行命令或操作，但在实际应用中较少使用。
                            • <mark>：表示文本中的高亮部分，通常用于标记需要引起注意的文本内容，在视觉上会以黄色背景等醒目方式显示。
                            • <ruby>：用于定义 Ruby 注释（中文注音或字符），常用于东亚文本中为字符提供注释或发音信息。
                            • <rt>：定义 Ruby 注释中与文本对应的部分。
                            • <rp>：定义 Ruby 注释中用于浏览器不支持 Ruby 注释时显示的括号部分。
                            • <time>：定义日期或时间，可以包含日期、时间、年份、月份等信息，有助于机器解析和处理时间数据。

                            # 语义化面试

                            面试高频点：为什么要用语义化标签？

                            • 有利于SEO
                            • 提高页面可读性、可维护性
                            • Assistive Technology 更易解析页面结构

                            语义化相关概念

                            • 面试官 ：你如何理解“语义化”？在前端开发中，它有什么重要性？
                            • 回答思路 ：语义化是指在编写 HTML 代码时，使用具有明确语义的 HTML 标签来组织和标记页面内容，使网页的结构和内容含义更加清晰、明确，让人和机器（如搜索引擎、屏幕阅读器等）都能更容易地理解和处理网页信息。它的重要性在于提高网页的可读性、可维护性、可访问性和搜索引擎优化（SEO）效果。

                            实现语义化的具体方法

                            • 面试官 ：你会如何在实际的项目中实现语义化？
                            • 回答思路 ：首先，我会根据页面内容的结构和含义选择合适的 HTML5 语义化标签，如 <header>、<nav>、<main>、<article>、<section>、<aside>、<footer> 等。其次，我会遵循语义化的层次结构，合理使用标题标签 <h1> - <h6>，确保页面内容的层次分明。此外，我还会合理使用语义化的行内元素，如 <strong>、<em>、<mark> 等来强调文本内容。

                            语义化对比非语义化

                            • 面试官 ：请举例说明语义化和非语义化代码的区别和优势？
                            • 回答思路 ：非语义化代码通常使用大量的 <div> 和 <span> 元素来构建页面结构，这些元素本身没有明确的语义含义，例如：<div class="header">、<div class="nav"> 等。而语义化代码则使用具有明确语义的 HTML5 标签，如 <header>、<nav> 等。语义化代码的优势在于：代码可读性更好，其他开发者更容易理解代码的意图；对搜索引擎更友好，有助于提高 SEO；对屏幕阅读器等辅助设备更友好，提升网页的可访问性。

                            语义化在实际工作中的应用

                            • 面试官 ：在实际工作中，你遇到过哪些语义化相关的挑战？你是如何解决的？
                            • 回答思路 ：在实际项目中，有时会遇到老项目代码非语义化严重，需要逐步重构的情况。我会先对现有代码进行评估，识别出非语义化的部分，然后在合适的时机（如功能迭代、页面改版等）逐步替换为语义化标签。在使用语义化标签时，我也遇到过浏览器兼容性问题，如某些旧版浏览器对 HTML5 新标签支持不佳，我会通过添加 Polyfill [待完成学习]或使用 CSS 自定义样式来确保兼容性。

                            # 链接

                            1. <a> 标签

                              1. 基本功能 ：<a> 标签用于创建超链接，允许用户点击链接跳转到其他网页、页面内的特定位置、下载资源或发送邮件等。
                              2. href 属性 ：
                                • 页面跳转 ：指定链接的目标 URL，可以是绝对路径或相对路径。例如 <a href="https://www.example.com">链接到外部网站</a> 表示链接到指定的外部网站；<a href="/about.html">链接到网站内的页面</a> 表示链接到网站内的某个页面。
                                • 页面内跳转 ：通过指定页面内的锚点来实现页面内跳转。例如 <a href="#section1">跳转到页面内的 section1</a>，页面内需要有一个对应的目标元素 <h2 id="section1">section1</h2>。
                                • 下载资源 ：<a href="https://www.example.com/file.pdf" download>下载文件</a> 可以引导浏览器下载指定的资源，而非直接在浏览器中打开。
                              3. target 属性 ：指定链接打开的方式。
                                • _blank ：在新窗口或标签页中打开链接。
                                • _self ：在当前窗口或标签页中打开链接（默认行为）。
                                • _parent ：在父级窗口或标签页中打开链接，常用于嵌套的 iframe 场景。
                                • _top ：在整个窗口中打开链接，会移除所有 iframe。

                            2. <link> 标签

                              1. 基本功能 ：<link> 标签用于定义文档与外部资源之间的关系，通常放在 HTML 文档的 <head> 部分。
                              2. 引用 CSS 文件 ：通过 <link rel="stylesheet" href="styles.css"> 可以将外部的 CSS 文件链接到 HTML 文档中，从而实现页面样式的统一管理。
                              3. 引用图标 ：可以用于定义网页的图标，例如 <link rel="icon" href="favicon.ico"> 可以设置网页标签栏显示的图标。
                              4. 其他用途 ：还可以用于指定其他资源的链接关系，如 <link rel="alternate" href="https://www.example.com/feed/rss" type="application/rss+xml" title="RSS Feed"> 可以指定 RSS 订阅源。

                            # 表单

                            • <form> ：用于定义表单，包含一组交互项供用户填写提交，常见于用户登录、注册或搜索功能等场景。
                            • <input> ：可定义多种类型的用户输入元素，如文本、密码、按钮等，常用于各种数据输入框的创建。
                            • <button> ：用于定义点击按钮以实现交互，可用于提交表单或执行其他操作。
                            • <label> ：为输入项定义标签，能够改善表单的可读性和用户体验，起到标示表单元素含义的作用。
                            • <textarea> ：定义多行文本输入框，适用于用户填写详细描述或留言内容等情况。
                            • <select> ：用于定义下拉选择框，包含多个选项，常用于让用户选择数据项。

                            # 表单校验

                            HTML5 内置校验机制

                            1. 针对 <input> 元素
                              • required 属性 ：
                                • 逻辑：当为 <input> 元素设置了 required 属性时，表单提交前会检查该输入框是否为空。如果为空，浏览器会阻止表单提交，并通常会显示一个默认的错误提示信息，如 “请填写此字段”。
                                • 底层逻辑示例：假设有一个带有 required 属性的文本输入框 <input type="text" required>，在表单提交事件触发时，浏览器的表单校验 API 会检查该输入框的值。如果值为 ""（空字符串），就会激活校验失败流程，阻止表单的默认提交行为，并根据浏览器的实现显示相应的错误提示。
                              • type 属性校验 ：
                                • 逻辑：不同类型的 <input> 元素会对输入内容进行类型校验。例如，当 type="email" 时，会检查输入内容是否符合电子邮件地址的格式；type="number" 则会验证输入是否为数字。
                                • 底层逻辑示例：以 <input type="email"> 为例，浏览器内部有一套电子邮件格式的正则表达式规则（例如，简单的规则可能类似 /^[^\s@]+@[^\s@]+\.[^\s@]+$/）。在表单校验时，会用这个正则表达式去匹配输入框的值。如果匹配失败，就像用户输入了 “invalidemail”（缺少 @ 符号），表单校验就会失败，阻止提交并提示类似 “请输入有效的电子邮件地址” 的信息。
                              • pattern 属性 ：
                                • 逻辑：通过 pattern 属性可以自定义输入内容的正则表达式校验规则。例如，<input type="text" pattern="[A-Za-z]{3}" required> 表示要求用户输入恰好 3 个英文字母。
                                • 底层逻辑：在表单校验时，浏览器会获取输入框的值，然后用指定的正则表达式进行匹配。如果匹配不成功，就会阻止表单提交并提示错误信息，如 “输入不符合要求的格式”。
                              • min 和 max 属性（适用于数字类型输入框） ：
                                • 逻辑：对于 type="number" 或 type="range" 等数字类型的输入框，min 和 max 属性可以限制输入的数字范围。如果输入的数字小于 min 或大于 max，校验就会失败。
                                • 底层逻辑示例：对于 <input type="number" min="1" max="10">，当用户输入 11 时，浏览器在表单校验时会比较输入的值与 min 和 max 设定的范围。如果超出范围，就会阻止表单提交并提示类似 “请输入介于 1 和 10 之间的数字” 的错误信息。
                            2. 针对 <input type="checkbox"> 和 <input type="radio"> 元素
                              • required 属性校验 ：
                                • 逻辑：对于单选按钮组（<input type="radio">）或复选框组（<input type="checkbox">），当设置了 required 属性时，会检查是否至少选中了一个选项。如果没有选中，表单提交就会被阻止。
                                • 底层逻辑示例：假设有一组单选按钮，它们共享相同的 name 属性，并且其中一个 <input type="radio" name="choice" required>。在表单提交时，浏览器会检查这个单选按钮组中是否有选中的选项。如果没有选中的选项，浏览器会阻止表单提交并提示类似 “请选择一个选项” 的错误信息。

                            CSS 伪类在表单校验中的辅助作用

                            • :valid 和 :invalid 伪类
                              • 逻辑：:valid 伪类用于选择通过校验的表单元素，:invalid 伪类用于选择校验失败的表单元素。通过 CSS 样式，可以为不同校验状态的表单元素提供视觉反馈。例如，可以给通过校验的输入框添加绿色边框，给校验失败的输入框添加红色边框。
                              • 底层逻辑示例：在 CSS 中，你可以这样设置：
                            input:valid {
  border: 2px solid green;
}

input:invalid {
  border: 2px solid red;
}

                            当表单元素校验状态改变时，浏览器会根据校验结果自动应用这些 CSS 样式规则。例如，当一个带有 required 属性的输入框内容有效（不为空）时，它会被匹配到 :valid 伪类，从而应用绿色边框样式；如果内容为空（校验失败），就会应用红色边框样式。

                            # 表格

                            • <table> 元素 ：
                            • 表格标题和表头 ：
                              • <caption> 元素 ：用于为表格添加标题，放置在 <table> 元素的开头，表格标题通常位于表格的顶部或底部，为表格提供简短的描述。
                              • <th> 元素 ：定义表格的表头单元格，默认情况下，表头单元格中的内容会以居中加粗的方式显示，用于标识表格中每一列或每一行的数据含义。
                            • 表格分组 ：
                              • <thead>、<tbody>、<tfoot> 元素 ：分别用于定义表格的头部、主体和底部，用于对表格行进行分组，有助于提高表格的可读性、可维护性，也对表格的样式设置和打印控制等有重要作用。在实际使用中，通常把表格的标题行放在 <thead> 中，表格的数据主体行放在 <tbody> 中，表格的底部总结行放在 <tfoot> 中。
                              • <colgroup> 和 <col> 元素 ：<colgroup> 用于对表格的列进行组合，<col> 定义 <colgroup> 中的一列，可为多列同时设置相同的属性，如宽度、对齐方式等。
                            • 单元格合并 ：
                              • rowspan 属性 ：用于指定一个单元格跨多行显示，其属性值为该单元格跨越的行数，当单元格跨行时，它所跨越的下方单元格将不再显示。
                              • colspan 属性 ：用于指定一个单元格跨多列显示，属性值为该单元格跨越的列数，跨列时，它所跨越的右侧单元格将不再显示。
                            • 表格样式设置 ：
                              • 边框样式 ：可以通过 CSS 的 border 属性为表格、单元格设置边框样式，如边框的宽度、颜色、样式（实线、虚线、点线等）。
                              • 单元格内数据对齐方式 ：使用 CSS 的 text-align（水平对齐）、vertical-align（垂直对齐）属性来设置单元格内数据的对齐方式。
                              • 表格宽度和高度 ：既可以通过 HTML 的 width、height 属性设置表格的宽度和高度，也可以使用 CSS 的 width、height 属性进行更灵活的设置。
                            • 表格的可访问性和语义化 ：
                              • 关联表头和数据单元格 ：在复杂的表格中，可以通过为数据单元格（<td>）设置 headers 属性，指定该单元格对应的一个或多个表头（<th>）的 id 值，以加强表格数据与表头之间的关联性，提高表格的可访问性。
                              • 使用语义化的 <th>、<td> 元素 ：合理使用 <th> 表头元素和 <td> 数据单元格元素，使表格具有明确的语义，有助于屏幕阅读器等辅助设备正确解析表格内容，为有特殊需求的用户提供必要的信

                            # 总结（Summary）

                            ​ 在 HTML 中，可以通过一些特定的元素和属性对内容进行归纳概括。例如，<summary> 元素可为 <details> 元素提供概括性标题，点击该标题可显示或隐藏详细内容。此外，在文档中合理使用标题元素（<h1>-<h6>）也能对内容进行层次分明的归纳总结，帮助用户快速了解页面结构和主题。

                            # 实体（Entities）

                            ​ 用于定义文档中的特殊字符或符号。在 HTML 中，一些字符具有特殊意义，如小于号（<）、大于号（>）、 amp;符号（&）等，直接使用可能会导致代码错误或误解。因此，需要使用实体名称或编号来表示这些特殊字符，例如 &lt; 表示小于号，&gt; 表示大于号，&amp; 表示 amp;符号。此外，还可以使用实体来插入一些无法直接从键盘输入的特殊符号，如版权符号（© &copy;）、商标符号（™ &trade;）等。

                            # 命名空间（Namespaces）

                            ​ 在 XML 及其相关技术（如 XHTML）中，命名空间用于确保元素名称的唯一性。通过为元素定义命名空间，可以避免不同来源或不同用途的元素名称冲突。在 HTML5 中，虽然引入了对 XML 命名空间的支持，但在实际使用中，对于大多数常规的 HTML 元素，通常不需要显式指定命名空间，因为 HTML5 的松散语法允许直接使用元素名称。不过，在一些特定场景下，例如当 HTML 文档中嵌入了来自其他命名空间的元素（如 SVG、MathML 等），则需要使用命名空间来区分不同来源的元素。命名空间通过 URI（统一资源标识符）来标识，例如，SVG 的命名空间为 http://www.w3.org/2000/svg，MathML 的命名空间为 http://www.w3.org/1998/Math/MathML。

                            # Canvas 和 SVG 的区别与场景？

                            # 一、定义概述

                            • Canvas是 HTML5 提供的位图画布，用于动态绘制像素级的2D图形（可通过 JS 实时绘制、操作）。
                            • **SVG（Scalable Vector Graphics）**是一种基于 XML 的矢量图形描述语言，可直接在 HTML 中用标签嵌套和操作。

                            # 二、主要区别

                            对比项 Canvas SVG
                            图像类型 位图（像素级） 矢量图（由图形/路径对象构成）
                            操作方式 通过 JavaScript 绘制（逐像素渲染） HTML 标签结构和 CSS 样式/JS 可操作
                            事件支持 不支持图形对象级别事件 支持每个对象/元素的事件绑定
                            动态交互 需手动管理每个对象状况，复杂 每个元素都可独立操作，方便动态交互
                            放大缩小 会失真、马赛克 无损缩放，始终保持清晰
                            渲染性能 处理大量对象/高频更新时更高效 复杂场景下性能受限
                            文件大小 复杂图像时文件较大 简单图形时文件小，复杂图形时变大
                            导出 可转为图片（如 PNG、JPG） 可直接保存为 SVG 文件
                            代码层次 操作具体像素、无图形对象层级 具有 DOM 层次结构，可直接查找和操作

                            # 三、应用场景

                            • Canvas 适合：

                              • 动态游戏、动画
                              • 实时数据可视化（如波浪、粒子等特效）
                              • 图像处理与像素级运算（如图片合成、滤镜）
                              • 频繁重绘的大数据量渲染

                            • SVG 适合：

                              • 图标、Logo、图示等需要无损缩放的图形
                              • 静态或交互性强的可视化（如流程图、地图、可点击的统计图）
                              • 结构分明、元素少但层次复杂的场景
                              • 字体轮廓、自适应高分屏界面

                            # 四、典型代码对比

                            • Canvas 示例：

                              • <canvas id="myCanvas" width="200" height="100"></canvas>
<script>
  const ctx = document.getElementById('myCanvas').getContext('2d');
  ctx.fillStyle = 'red';
  ctx.fillRect(10, 10, 100, 50);
</script>

                            • SVG 示例：

                              • <svg width="200" height="100">
  <rect x="10" y="10" width="100" height="50" fill="red"></rect>
</svg>

                            # 五、答题小结（面试模板）

                            Canvas 是基于像素的画布，适合高性能的大量渲染和实时动画、图像处理等场景，操作灵活但事件绑定和对象管理较复杂。SVG 是基于矢量、DOM结构的标记语言，适合需要无损缩放、高度交互和结构清晰的图形，比如图标、流程图、统计图表等，两者应根据场景和性能需求合理选择。
