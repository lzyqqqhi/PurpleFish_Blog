      # Vue 事件修饰符 & 性能优化速记

      适用版本：Vue 3.x
      作用：用“模板声明”替代“方法逻辑”，提升可读性与运行时性能。

      # 1. 常用事件修饰符一览

      修饰符 功能 模板示例 等价 JS
      .stop 阻止冒泡 @click.stop="fn" e.stopPropagation()
      .prevent 阻止默认行为 @submit.prevent="fn" e.preventDefault()
      .capture 捕获阶段触发 @click.capture="fn" addEventListener('click', fn, true)
      .self 仅自身触发 @click.self="fn" if (e.target !== e.currentTarget) return
      .once 只触发一次 @click.once="fn" 自动解绑
      .passive 永不阻止默认，提升滚动性能 @scroll.passive="fn" { passive: true }

      .throttle 不是内置修饰符，需要自定义指令或 lodash。

      ──────────────────── 一、v-model 多值绑定（Vue 3.4+ 新特性）

      1. 语法 <input v-model="a" v-model="b"> • 同一标签可同时绑定多个 v-model，实际等价于： <input :value="a" @input="a=$event.target.value" :value="b" @input="b=$event.target.value"> • 仅对原生 <input>、<textarea>、<select> 生效。
      2. 使用场景 • 双向绑定多个简单值，避免额外包裹对象。 • 注意：顺序决定绑定优先级；同名属性后写覆盖先写。

        app.directive('throttle', {
  mounted(el, { value, arg = 200 }) {
    el._fn = throttle(value, Number(arg))
    el.addEventListener('input', el._fn)
  },
  unmounted(el) {
    el.removeEventListener('input', el._fn)
  }
})

      2. 模板使用：<input v-throttle:500="onFastInput">

      # Vue 事件修饰符 & 性能优化速记

> 适用版本：Vue 3.x  
> 作用：用“模板声明”替代“方法逻辑”，提升可读性与运行时性能。

---

## 1. 常用事件修饰符一览

| 修饰符 | 功能 | 模板示例 | 等价 JS |
|--------|------|-----------|---------|
| `.stop` | 阻止冒泡 | `@click.stop="fn"` | `e.stopPropagation()` |
| `.prevent` | 阻止默认行为 | `@submit.prevent="fn"` | `e.preventDefault()` |
| `.capture` | 捕获阶段触发 | `@click.capture="fn"` | `addEventListener('click', fn, true)` |
| `.self` | 仅自身触发 | `@click.self="fn"` | `if (e.target !== e.currentTarget) return` |
| `.once` | 只触发一次 | `@click.once="fn"` | 自动解绑 |
| `.passive` | 永不阻止默认，提升滚动性能 | `@scroll.passive="fn"` | `{ passive: true }` |

> `.throttle` 不是内置修饰符，需要自定义指令或 lodash。

      # 3. 高频场景优化

      场景 推荐做法 收益
      滚动/触摸 @scroll.passive="fn" 移动端 FPS ↑，避免卡顿
      输入实时搜索 v-throttle 自定义指令 减少请求数，降低服务器压力
      一次性弹窗 @click.once="openModal" 无需手动解绑，内存友好

      # 优化相关概念

      • 可读性: 使用修饰符可以减少方法内部的逻辑，使代码更加简洁和易于理解。例如，使用 .prevent 可以直接在模板中表明意图。
      • 性能: 通过限制事件的触发频率（如使用 .throttle），可以提高性能，尤其是在处理频繁触发的事件（如滚动或输入）时。
      • 组合使用: 事件修饰符可以组合使用。例如，可以同时使用 .stop 和 .prevent：
      <button @click.stop.prevent="handleClick">Click Me</button>
      • 自定义修饰符: 虽然 Vue 3 主要提供了内置的事件修饰符，但也可以通过自定义指令实现类似功能。
