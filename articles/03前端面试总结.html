                # HTML5 新增了哪些语义化标签？有何意义？

                详细：html

                一、HTML5 新增语义化标签一览
标签语义/用途

                # 如何实现本地存储？与 cookie 区别？

                # 一、本地存储的实现方式

                详细：本地存储与cookie

                HTML5 提供了两种主流本地存储方案：

                1. localStorage
                  1. 特点：持久存储，除非手动清除，否则数据永久存在。
                  2. 用法示例： 
                    • // 保存数据
localStorage.setItem('key', 'value');
// 获取数据
const value = localStorage.getItem('key');
// 删除数据
localStorage.removeItem('key');
// 清空所有数据
localStorage.clear();
                2. sessionStorage
                  1. 特点：仅在当前会话（浏览器窗口/标签页）有效，关闭窗口即被清除。
                  2. 用法与 localStorage 相同。
                3. IndexedDB (用于结构化和大量数据，本题通常答到 localStorage 和 sessionStorage 即可)

                # 二、本地存储与 cookie 的区别

                对比点 localStorage / sessionStorage cookie
                储存大小 约 5MB（各浏览器略有差异） 约 4KB
                生命周期 localStorage 永久
                sessionStorage 会话                		 可设置过期时间
                与 HTTP 请求 不会自动随请求发送 每次请求会自动携带
                操作接口 JS API，简洁易用 需要手动字符串拼接或用库
                用途 前端数据缓存，增强型离线应用等 通常用于会话管理、登录信息等
                安全性 相对更安全（同源策略限制访问） 易被抓包、劫持，有安全风险
                服务端能否操作 否（只能前端读写） 是（服务器通过 Set-Cookie 设置）

                # 三、总结面试标准答法

                本地存储可以通过 HTML5 的 localStorage 和 sessionStorage 来实现，无需与服务器交互，适合大容量、长期或会话级的数据保存。与 cookie 相比，本地存储容量更大，不会自动随每次 HTTP 请求携带，安全性更高，使用也更加简单。cookie 通常用于服务端会话管理，容量小、每次请求自动发送，并可以由后端设置。

                # 四、面试简要要点

                • localStorage/SessionStorage 容量大，接口简单，不随请求，前端操作
                • cookie 容量小，随请求自动发送，可由服务器生成，主要用于会话等
                • 不能用 localStorage 代替 cookie 做登录凭证，否则有安全隐患

                # 如何嵌入音视频？需要注意哪些兼容性问题？

                # 一、HTML5 嵌入音视频的方式

                详细：html嵌入音视频

                HTML5 提供了原生的音频和视频标签：

                • 音频嵌入

                  • <audio src="music.mp3" controls autoplay loop></audio>

                  • 常用属性：

                  • src ：音频文件路径

                  • controls ：显示播放控件

                  • autoplay ：自动播放（注意：大部分现代浏览器已限制自动播放，通常需静音或用户交互触发）

                  • loop ：循环播放

                  • muted ：静音播放

                • 视频嵌入

                  • <video src="movie.mp4" controls poster="thumb.jpg" width="480"></video>

                  • 常用属性：

                  • src ：视频文件路径

                  • controls ：显示播放控件

                  • poster ：视频加载前显示的缩略图

                  • width/height ：宽高尺寸

                  • autoplay、loop、muted 等同音频

                • 多格式兼容写法

                  • <video controls>
  <source src="movie.mp4" type="video/mp4">
  <source src="movie.webm" type="video/webm">
  <source src="movie.ogv" type="video/ogg">
  浏览器不支持 video 标签。
</video>

                # 二、兼容性问题与注意事项

                1. 格式兼容性

                  1. 各浏览器支持的音视频格式不同。主流建议：
                    • 视频优先提供 mp4 (H.264)、webm 和 ogg 格式
                    • 音频优先提供 mp3、ogg 和 wav 格式

                2. 自动播放限制

                  1. 现代浏览器通常禁止自动播放有声音的音视频，需 muted 或用户交互触发

                3. 移动端兼容性

                  1. 移动浏览器（如微信、iOS Safari）对 autoplay、controls 等支持各有差别，需要实机测试

                4. 降级方案

                  1. 可以在 <audio> 或 <video> 标签内加入文本内容，供不支持 HTML5 的老浏览器显示
                  2. 传统 <object> 或 Flash 播放器作为备选，但现代开发已很少使用

                5. 字幕和无障碍

                  1. 利用 <track> 子标签添加字幕，便于国际化和无障碍浏览 
                    • <video controls>
  <source src="movie.mp4" type="video/mp4">
  <track src="movie.vtt" kind="subtitles" srclang="en" label="English">
</video>

                6. 跨域资源

                  1. 从第三方服务器加载音视频时，要注意 CORS 跨域策略设置，避免播放失败

                # 三、面试标准回答范例

                HTML5 通过 <audio> 和 <video> 标签可以方便地嵌入音视频。需要注意的是，不同浏览器对音视频格式的支持并不完全一致，因此建议提供多种格式以保证兼容性。同时，现代浏览器对自动播放有较严格的限制，通常需设置 muted 或依赖用户操作才能播放。此外，还需要关注移动端的特殊行为和 CORS 跨域设置等，以确保在各种终端和场景下都能顺利播放音视频内容。

                # Canvas 和 SVG 的区别与场景？

                # 一、定义概述

                • Canvas是 HTML5 提供的位图画布，用于动态绘制像素级的2D图形（可通过 JS 实时绘制、操作）。
                • **SVG（Scalable Vector Graphics）**是一种基于 XML 的矢量图形描述语言，可直接在 HTML 中用标签嵌套和操作。

                # 二、主要区别

                对比项 Canvas SVG
                图像类型 位图（像素级） 矢量图（由图形/路径对象构成）
                操作方式 通过 JavaScript 绘制（逐像素渲染） HTML 标签结构和 CSS 样式/JS 可操作
                事件支持 不支持图形对象级别事件 支持每个对象/元素的事件绑定
                动态交互 需手动管理每个对象状况，复杂 每个元素都可独立操作，方便动态交互
                放大缩小 会失真、马赛克 无损缩放，始终保持清晰
                渲染性能 处理大量对象/高频更新时更高效 复杂场景下性能受限
                文件大小 复杂图像时文件较大 简单图形时文件小，复杂图形时变大
                导出 可转为图片（如 PNG、JPG） 可直接保存为 SVG 文件
                代码层次 操作具体像素、无图形对象层级 具有 DOM 层次结构，可直接查找和操作

                # 三、应用场景

                • Canvas 适合：

                  • 动态游戏、动画
                  • 实时数据可视化（如波浪、粒子等特效）
                  • 图像处理与像素级运算（如图片合成、滤镜）
                  • 频繁重绘的大数据量渲染

                • SVG 适合：

                  • 图标、Logo、图示等需要无损缩放的图形
                  • 静态或交互性强的可视化（如流程图、地图、可点击的统计图）
                  • 结构分明、元素少但层次复杂的场景
                  • 字体轮廓、自适应高分屏界面

                # 四、典型代码对比

                • Canvas 示例：

                  • <canvas id="myCanvas" width="200" height="100"></canvas>
<script>
  const ctx = document.getElementById('myCanvas').getContext('2d');
  ctx.fillStyle = 'red';
  ctx.fillRect(10, 10, 100, 50);
</script>

                • SVG 示例：

                  • <svg width="200" height="100">
  <rect x="10" y="10" width="100" height="50" fill="red"></rect>
</svg>

                # 五、答题小结（面试模板）

                Canvas 是基于像素的画布，适合高性能的大量渲染和实时动画、图像处理等场景，操作灵活但事件绑定和对象管理较复杂。SVG 是基于矢量、DOM结构的标记语言，适合需要无损缩放、高度交互和结构清晰的图形，比如图标、流程图、统计图表等，两者应根据场景和性能需求合理选择。

                # 浏览器如何实现跨域资源共享？

                详细：跨域

                # 一、什么是跨域

                • 跨域（Cross-Origin） 指的是浏览器出于安全策略“同源策略”（Same-Origin Policy）限制，不允许一个域下的 JS 访问另一个域的资源或数据。
                • 同源 指协议、域名、端口均一致。否则就是“跨域”。

                # 二、实现跨域资源共享的方法

                # 1. CORS（Cross-Origin Resource Sharing，跨域资源共享）- 主流且标准方法 常见请求头 响应头
                • 原理：服务器响应时设置特定 HTTP 头，允许指定源的网页访问本资源。
                • 核心 HTTP 响应头：
                  • Access-Control-Allow-Origin：允许的域名
                  • Access-Control-Allow-Methods：允许的方法(GET、POST等)
                  • Access-Control-Allow-Headers：允许携带的自定义头
                • 示例： 
                  • Access-Control-Allow-Origin: https://example.com
                • 流程：
                  • 浏览器发起跨域请求，服务器若带上正确的 CORS 响应头，浏览器则允许前端访问响应内容，否则拦截（前端拿不到响应数据）。
                # 2. JSONP（仅支持GET，老旧方式）
                • 利用 <script> 标签不受同源策略限制，通过回调函数获取数据。 const res = loadData()
                • 仅适合兼容性要求高、只用 GET 请求的场景。
